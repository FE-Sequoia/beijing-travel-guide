# 北京慢游记小程序：人文杂志感视觉与排版优化设计

## 目标

在不改变本地静态数据、搜索筛选、收藏、阅读历史、行程等既有能力的前提下，统一小程序的视觉语言，并让长篇攻略和景点详情更易读、更有节奏。

## 设计方向

采用「京味人文杂志感」：克制、温暖、有纸张质感。背景使用低饱和米白，内容承载使用暖白，墨绿用于主要操作与分类识别，朱砂只用于收藏、提醒和少量重点信息。避免同时出现过多色块、标签和强阴影。

## 视觉令牌

全局样式定义可复用令牌，并由页面和组件消费：

- 背景：`#F7F3EC`；内容表面：`#FFFCF7`；弱表面：`#F1ECE3`。
- 主文字：`#263238`；辅助文字：`#6E746F`；正文：`#48524E`。
- 墨绿：`#1F5B4F`；浅绿：`#E4EFE9`；朱砂：`#B9513A`；浅朱砂：`#F8E8E2`。
- 页面水平内边距统一为 `32rpx`；常规区块间距为 `48rpx`；卡片内边距为 `24rpx`；卡片圆角为 `24rpx`。
- 阴影仅用于可点击卡片，改为低对比度描边与轻阴影；信息容器优先使用分隔线，而非层层套卡片。
- 标题、导语、正文、元信息分别使用固定字号与行高层级，正文不使用单行省略。

## 页面与组件

### 全局与共用组件

`app.wxss` 提供颜色、间距、表面、文本等基础样式。`section-title` 使用“标题 + 简短导语 + 文字操作”的固定结构，并增加轻量分隔。`place-card` 分为标准列表卡与紧凑推荐卡：图片比例、标题、摘要、标签数量和内边距一致；摘要最多两行，标签最多两枚，以避免列表噪声。

### 首页

首屏保留情绪化引导，但减少大面积渐变和无关装饰。六宫格分类作为明确入口；趣玩采用两列内容卡，行程灵感改为纵向摘要卡或横向仅保留一类视觉样式。每个模块都采用相同的标题间距和卡片节奏。

### 探索、攻略、行程、我的

列表页统一为：页标题与导语、单一搜索/筛选区、结果信息、纵向内容列表。筛选项改为低干扰胶囊按钮；搜索框与排序不再与内容卡竞争视觉重量。我的页面将收藏与历史视作两个内容区块，空状态保持简洁。

### 景点与攻略详情

详情页面使用一致的信息顺序：封面、分类眉题、主标题与收藏、导语、标签、关键信息、正文分节、关联内容。信息项改为易扫读的两列或纵向键值布局；正文段落限制阅读宽度，设置更舒适的行高、段落留白和章节间距。关联地点使用单行列表和细分隔线。攻略详情复用同一正文排版规则，避免与景点详情产生不同的阅读体验。

### 行程详情

行程头部保留主题信息，但缩小装饰面。每日安排使用清晰的 Day 标识与时间轴感布局；地点、时间、说明三者层级稳定，减少每个停靠点的卡片感。

## 数据与交互边界

不变更 `data/*.json`、`data/*.js` 和 `utils/data.js` 对外数据契约；页面继续消费现有字段。点击、搜索、筛选、收藏、历史记录、关联地点跳转和行程跳转的事件名称与目标页面保持不变。封面加载失败时使用统一的渐变占位样式。

## 可访问性与适配

保证正文、辅助文字及交互色具有可读对比度；可点击目标保留足够的内边距；所有容器设定 `min-width: 0` 并避免固定超宽元素，以消除横向溢出。图片与文字截断规则在窄屏下保持稳定。

## 验收与测试

- 既有 `data.test.js`、`storage.test.js`、`project.test.js` 全部通过。
- 增加针对 WXML/WXSS 的静态检查，覆盖全局视觉令牌、详情正文样式、列表卡摘要限制和禁止横向溢出的关键规则。
- 对所有小程序 JavaScript 执行 `node --check`，并运行 `git diff --check`。
- 在微信开发者工具中检查首页、探索、景点详情、攻略详情、行程详情和我的页面的窄屏展示；重点确认无横向溢出、正文层级清晰、点击功能不回归。

## 非目标

本次不迁移云开发、不替换静态数据源、不新增网络请求、不引入第三方 UI 框架，也不重做信息架构或新增页面。
